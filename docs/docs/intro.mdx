# Getting Started

WC-Aria is a framework-agnostic, accessible web components library built with performance and WCAG 2.1 AA compliance in mind.

## Installation

```bash
npm install waria
```

## Quick Start

```ts
import { App } from "waria";

// Configure and initialize
App.start({ hash: true });
```

All components are now registered and ready to use.

## Basic Usage

```html
<!-- Navigation with URL sync -->
<w-nav label="Main navigation">
  <a slot="item" href="/home">Home</a>
  <a slot="item" href="/about">About</a>
</w-nav>

<!-- View routing -->
<w-view path="/home">
  <h1>Home Page</h1>
</w-view>

<w-view path="/about">
  <h1>About Page</h1>
</w-view>
```

## Slot-Based Composition

All components use the `slot` attribute for composition:

```html
<w-dialog>
  <button slot="trigger">Open Dialog</button>
  <div slot="content">
    <h2>Dialog Title</h2>
    <p>Dialog content goes here.</p>
    <button slot="close">Close</button>
  </div>
</w-dialog>
```

Common slots used across components:

| Slot      | Purpose                                   |
| --------- | ----------------------------------------- |
| `trigger` | Element that activates the component      |
| `content` | Main content area                         |
| `item`    | Repeated items (menus, lists, navigation) |
| `close`   | Close button                              |
| `label`   | Label text                                |

## Event Handlers

Components emit custom events that you can listen to. Use the `w-*` attribute syntax for inline handlers:

```html
<w-dialog w-open="console.log('opened')" w-close="console.log('closed')">
  <button slot="trigger">Open</button>
  <div slot="content">...</div>
</w-dialog>

<w-menu w-select="handleSelect(event.detail)">
  <button slot="trigger">Menu</button>
  <div slot="content">
    <button slot="item" name="edit">Edit</button>
  </div>
</w-menu>
```

The `event` object is available in the handler with:

- `event.type` - Event name (e.g., "open", "close")
- `event.detail` - Event data (varies by component)
- `event.target` - The component element

You can also use `addEventListener`:

```js
document.querySelector('w-dialog').addEventListener('open', (e) => {
  console.log('Dialog opened');
});
```

## Keyboard Navigation

All components support full keyboard navigation:

- **Tab** - Move between focusable elements
- **Arrow Keys** - Navigate within components
- **Enter/Space** - Activate buttons and selections
- **Escape** - Close overlays and cancel operations
- **Home/End** - Jump to first/last items
